Automatic Method

Netzhaut uses a "bootstrap" library to resolve internal dependencies at runtime. This means that you only need to link to a tiny fraction of Netzhaut at compile time. You also dont have to necessarily compile anything yourself.

At runtime, Netzhaut will load internal libraries as needed. If a library is not present, Netzhaut can try to install it automatically by building it fresh from source. This behavior can be enabled and modified using [Nh_init] arguments.

TLDR for linux 1.Download header 2.Download libNetzhaut.so 3. Include Header 4. Pass automatic-method enabling arguments to [Nh_init] 5. Compile/Link to libNetzhaut.so 6. Execute.

Manual Method

Netzhaut uses an internal installer to build internal dependencies. The installer can be used to manually install all internal libraries and binaries.

INSTALLER USAGE


  NETZHAUT-INSTALLER USAGE

(BUILD)

  OPTIONS:
    -l [LIBRARIES...] Build LIBRARIES. If no LIBRARIES are specified, build all.
    -b [BINARIES...]  Build BINARIES. If no BINARIES are specified, build all.
    -i [FLAGS...]     Install FLAGS.
    -n [FLAGS...]     Suppress building and installing of FLAGS.

  FLAGS: 
    LIBRARIES: netzhaut  Module that applications link to.
               core      Core module that implements a broad array of functionality.
               network   Network module for communication with other devices.
               tty       Terminal module for command-line based interaction.
               io        IO module implementing WSI and Input Device handling.
               installer Installer module with auto-update capability.
               webidl    TODO

    BINARIES: webbrowser   TODO
              windowsystem TODO

  e.g. 
    -li            Build and install all libraries.
    -ib            Build and install all binaries.
    -lbi           Build and install all libraries and binaries.
    -im -et        Build all modules, examples and tools. Install modules.
    -ei webbrowser Build and install the 'webbrowser' example.
    -me core       Build the 'core' module and all examples.

  --flow    Activate flow logging capabilities. Only recommended for debugging purposes.
  --vulkan  Compile Vulkan shaders.
  --docs    Generate documentation. Requires doxygen (www.doxygen.nl)
  --offline Do not download any resources over network.

(OTHER)

ABOUT

Netzhaut runs web-technologies on the client-machine. You can use the library to write your own web-browser or anything else which makes use of web-technologies like HTML, CSS or JS.

Why use Netzhaut?

  • Fast HTML + CSS (W3C compliant) and JS (ECMA-262 + W3C compliant) execution
  • Fully fledged internal Window System
  • Easy to integrate and customize
  • Adaptive Threading based Architecture
  • Robust Network handling
  • Flexible Rendering using Vulkan, DirectX or Software Rendering
  • VR support
  • Extensive Logging capabilities
  • Automatic Updates
  • Few dependencies
  • Cross Platform
  • Written in C99
